## T023 - `MDVM` không dùng được → đổi sang `PPH_DBVM`

* **ID:** `T023`
* **Nội dung lỗi (エラー内容):** Khi đang lấy dữ liệu từ `MDVM`, trên S/4HANA phải đổi sang `PPH_DBVM` (MDVMからデータを取得している場合、PPH_DBVMに変更する).
* **Cách khắc phục (対応方法):** Đổi bảng `MDVM` → `PPH_DBVM`, phân giải `MDVM-MDKEY` thành 品目コード (MATNR) + プラント (WERKS), set điều kiện `PHANTOM` theo `MAFID`, và bổ sung các điều kiện cố định của `PPH_DBVM`.

### Nguyên tắc xử lý

* **Phân giải `MDVM-MDKEY`** (offset cố định):
  * 品目コード (material code) = ký tự thứ 4〜21 → `LV_MATNR = LV_MDKEY+3(18)`.
  * プラント (plant) = ký tự thứ 22〜25 → `LV_WERKS = LV_MDKEY+21(4)`.
* **`MDVM-MAFID` (record type) → `PPH_DBVM-PHANTOM`:**
  * `MAFID = 'D'` (ファントム / phantom) → điều kiện `PHANTOM = 'X'`.
  * Trường hợp khác (vd `'B'`) → `PHANTOM = ''`.
* **Điều kiện cố định của `PPH_DBVM`:**
  * `BERID` lấy cùng giá trị với `WERKS` → `BERID = LV_WERKS`.
  * `PLSCN = '000'`.
* **Xử lý SELECT-OPTION:**
  * Nếu điều kiện gốc là SELECT-OPTION (vd `S_MAFID`, `S_MDKEY`), bắt buộc thêm logic phân nhánh theo giá trị `OPTION`/`LOW`/`HIGH` (`LOOP AT S_... ` + `CASE`-`OPTION`) để suy ra giá trị scalar dùng cho điều kiện `=` của `PPH_DBVM`.
  * Với `S_MAFID`: `CASE OPTION` → `WHEN 'EQ'` → `CASE LOW` (`'D'` → phantom 'X', còn lại → ''); `WHEN 'BT'` → xử lý tương ứng range.
  * Với `S_MDKEY`: tương tự, lấy `LOW`/`HIGH` rồi phân giải MATNR/WERKS.
* Không tự đoán field mapping ngoài phần đã xác nhận; field gốc không có đích tương ứng trong `PPH_DBVM` (vd `GSAEN`, `ERDAT`) phải hỏi mapping/source trước khi sửa.

### Mapping điều kiện `MDVM` → `PPH_DBVM`

| ECC (`MDVM`) | S/4 (`PPH_DBVM`) | Ghi chú |
| --- | --- | --- |
| `MDVM` | `PPH_DBVM` | Đổi bảng. |
| `MDKEY` (ký tự 4〜21) | `MATNR` = `MDKEY+3(18)` | Phân giải 品目コード. |
| `MDKEY` (ký tự 22〜25) | `WERKS` = `MDKEY+21(4)` | Phân giải プラント. |
| `MAFID = 'D'` | `PHANTOM = 'X'` | Record type phantom. |
| `MAFID` (khác) | `PHANTOM = ''` | Không phải phantom. |
| (giá trị `WERKS`) | `BERID = WERKS` | Điều kiện `BERID` giống `WERKS`. |
| — | `PLSCN = '000'` | Điều kiện cố định. |
| `GSAEN` / `ERDAT` | ❓ | Không có trong pattern đã xác nhận — cần hỏi mapping. |

### Ví dụ — structure/table retrieval (構造取得の場合)

#### Cú pháp cũ (旧)

```abap
SELECT * FROM MDVM
  INTO TABLE IT_MDVM
  WHERE MAFID IN S_MAFID
    AND MDKEY IN S_MDKEY
    AND GSAEN IN S_GSAEN
    AND ERDAT IN S_ERDAT.
```

#### Cú pháp mới (新)

> Khi điều kiện là SELECT-OPTION thì thêm phân nhánh theo giá trị OPTION (SELECT-OPTIONの場合は、OPTIONの設定値による条件分岐処理を追加する).

```abap
LOOP AT S_MAFID INTO LS_MAFID.

  CASE LS_MAFID-OPTION.
    WHEN 'EQ'.
      CASE LS_MAFID-LOW.
        WHEN 'B'.
          LV_PHANTOM = ''.
        WHEN 'D'.                 " ファントムの場合
          LV_PHANTOM = 'X'.
        WHEN OTHERS.
      ENDCASE.
    WHEN 'BT'.
      "... xử lý range tương ứng
  ENDCASE.

  "... MDKEYがSELECT-OPTIONの場合は、同様に条件分岐処理を追加する
  LV_MDKEY = S_MDKEY-LOW.   " hoặc -HIGH tùy logic

  LV_MATNR = LV_MDKEY+3(18).      " 品目コードに分解
  LV_WERKS = LV_MDKEY+21(4).      " プラントに分解

  SELECT * FROM PPH_DBVM
    INTO TABLE IT_PPH_DBVM
    WHERE MATNR   = LV_MATNR
      AND WERKS   = LV_WERKS
      AND BERID   = LV_WERKS
      AND PLSCN   = '000'
      AND PHANTOM = LV_PHANTOM.

ENDLOOP.
```

> Nếu trước đó lưu vào internal table/structure kiểu `MDVM`, và logic phía sau vẫn dùng kiểu cũ, cần map lại field tương ứng (`MOVE-CORRESPONDING` hoặc gán theo field). Khi `SELECT *` nhưng cấu trúc `PPH_DBVM` khác `MDVM`, chỉ giữ field thật sự dùng.
