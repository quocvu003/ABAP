## T024 - Đối ứng hợp nhất bảng (テーブル統合対応) → đổi sang View tương ứng

* **ID:** `T024`
* **Nội dung lỗi (エラー内容):** Các bảng dưới đây bị hợp nhất/thay đổi cấu trúc trên S/4HANA, không lấy trực tiếp được nữa; cần lấy dữ liệu qua View tương ứng (テーブル統合対応).
* **Cách khắc phục (対応方法):** Đổi bảng nguồn trong câu lấy dữ liệu sang View tương ứng theo bảng mapping bên dưới.

### Nguyên tắc xử lý

* **Trường hợp lấy dữ liệu có chỉ định các trường cần lấy (項目指定あり):**
  → Chỉ cần đổi bảng nguồn sang **View tương ứng**. Giữ nguyên danh sách field và logic phía sau.
* **Trường hợp lấy toàn bộ các trường (`SELECT *` / 全項目取得):**
  → Đổi sang **View tương ứng**.
  → Lưu dữ liệu về internal table / structure kiểu cũ bằng **`MOVE-CORRESPONDING`** (vì cấu trúc View không hoàn toàn giống bảng gốc).
* Ưu tiên cú pháp ABAP mới, dùng host variable `@` trong Open SQL.
* Không tự đoán field mapping; nếu field cần dùng không có trong View, hỏi mapping/source trước khi sửa.

### Mapping bảng → View

| Bảng (ECC) | View (S/4HANA) |
| --- | --- |
| `COVP` | `V_COVP` |
| `ANEK` | `FAAV_ANEK` |
| `ANEP` | `FAAV_ANEP` |
| `ANEA` | `FAAV_ANEA` |
| `ANLC` | `FAAV_ANLC` |
| `ANLP` | `FAAV_ANLP` |

### Trường hợp 1 — Lấy dữ liệu có chỉ định field (項目指定あり)

Chỉ đổi bảng nguồn sang View, giữ nguyên field list.

#### Cú pháp cũ (旧)

```abap
SELECT BELNR
       BUZEI
       WRBTR
  FROM ANEP
  INTO TABLE LT_ANEP
  WHERE BUKRS = LV_BUKRS
    AND ANLN1 = LV_ANLN1.
```

#### Cú pháp mới (新)

```abap
SELECT BELNR
       BUZEI
       WRBTR
* S/4HANA Migration UPD START
*  FROM ANEP
  FROM FAAV_ANEP
* S/4HANA Migration UPD END
  INTO TABLE LT_ANEP
  WHERE BUKRS = LV_BUKRS
    AND ANLN1 = LV_ANLN1.
```

### Trường hợp 2 — Lấy toàn bộ field (`SELECT *` / 全項目取得)

Đổi sang View, lấy vào work area/table của View rồi `MOVE-CORRESPONDING` về structure/table kiểu cũ.

#### Cú pháp cũ (旧)

```abap
SELECT *
  FROM ANEP
  INTO TABLE LT_ANEP
  WHERE BUKRS = LV_BUKRS
    AND ANLN1 = LV_ANLN1.
```

#### Cú pháp mới (新)

```abap
* S/4HANA Migration UPD START
*SELECT *
*  FROM ANEP
*  INTO TABLE LT_ANEP
*  WHERE BUKRS = LV_BUKRS
*    AND ANLN1 = LV_ANLN1.
SELECT *
  FROM FAAV_ANEP
  INTO TABLE @DATA(LT_FAAV_ANEP)
  WHERE BUKRS = @LV_BUKRS
    AND ANLN1 = @LV_ANLN1.

LOOP AT LT_FAAV_ANEP INTO DATA(LS_FAAV_ANEP).
  MOVE-CORRESPONDING LS_FAAV_ANEP TO LS_ANEP.
  APPEND LS_ANEP TO LT_ANEP.
ENDLOOP.
* S/4HANA Migration UPD END
```

> Nếu logic phía sau vẫn dùng internal table/structure kiểu bảng cũ (`ANEP`, `COVP`, ...), bắt buộc map lại bằng `MOVE-CORRESPONDING`. Chỉ những field trùng tên giữa View và bảng cũ mới được chuyển; nếu thiếu field cần dùng, hỏi mapping trước khi sửa.
