## T022 - Index table (`VAPMA`/`VRPMA` 等) không dùng được → lấy từ bảng gốc + JOIN

* **ID:** `T022`
* **Nội dung lỗi (エラー内容):** Các index table như `VBRP_VRPMA` 等 không còn sử dụng được trên S/4HANA (S4では使用不可).
* **Cách khắc phục (対応方法):** Đổi sang lấy dữ liệu từ **bảng gốc của index** (インデックスの元テーブル). Khi đang lấy đồng thời cả thông tin header và明細 (detail) thì **kết hợp (JOIN) header và明細** để lấy. Cơ bản tên field giống index, nhưng có một số field đổi tên (xem bảng).

### Nguyên tắc xử lý

* Index table → đổi sang **bảng gốc**. Field của `VBRP_VRPMA` được phân bổ sang tối đa 4 bảng gốc:
  * Phía đơn hàng: `VBAK` (header) + `VBAP` (明細).
  * Phía hóa đơn: `VBRK` (header) + `VBRP` (明細).
* Khi cần cả header và明細 cùng lúc → dùng **`INNER JOIN`** trên `VBELN`:
  * `VBAP~VBELN = VBAK~VBELN`
  * `VBRP~VBELN = VBRK~VBELN`
* Chỉ lấy các bảng gốc thật sự cần theo field đang dùng; không JOIN thừa bảng không dùng field nào.
* Tên field về cơ bản **giống index**; chỉ đổi với field đã xác nhận trong bảng mapping (vd `KUNNR` của index → `VBRK-KUNRG`).
* Nếu code đang lưu vào structure/table **kiểu index** (vd `TYPE VBRP_VRPMA`, `IS_VBRP_VRPMA`), phải đổi type và các field reference sang bảng gốc tương ứng (vd `IS_VBRP_VBRK`, `...-KUNRG`).
* Nếu code cũ lấy **toàn bộ field** (`SELECT *`), khi chuyển sang JOIN phải **chỉ lấy các field thật sự dùng** (使用する項目に絞って取得).
* Không tự đoán field mapping ngoài bảng đã xác nhận; nếu thiếu field cần dùng, hỏi mapping trước khi sửa.

### Ví dụ — lấy dữ liệu từ index + JOIN header/detail

#### Cú pháp cũ (旧)

```abap
SELECT *
  INTO LS_VBRP_VRPMA
  FROM VBRP_VRPMA
  WHERE MATNR = LS_MATERIAL-MATNR
    AND FKDAT BETWEEN F_DAY AND E_DAY.
```

#### Cú pháp mới (新)

> Nếu đang lưu vào kiểu index thì phải đổi type. Nếu trước đó lấy toàn bộ field thì chỉ lấy các field thật sự dùng (インデックステーブルの型に格納している場合は変更する / 全項目取得している場合は、使用する項目に絞って取得する).

```abap
SELECT KUNAG, KUNRG
  INTO LS_VBRP_VBRK
  FROM VBRP
  INNER JOIN VBRK
  ON VBRP~VBELN = VBRK~VBELN
  WHERE MATNR = LS_MATERIAL-MATNR
    AND FKDAT BETWEEN F_DAY AND E_DAY.
```

### Ví dụ — sửa reference structure index sang bảng gốc

#### Cú pháp cũ (旧)

```abap
SELECT SINGLE NAME1
  INTO OS_OUT-AGNAM
  FROM KNA1
  WHERE KUNNR EQ IS_VBRP_VRPMA-KUNAG.

OS_OUT-KUNRG = IS_VBRP_VRPMA-KUNNR.
OS_OUT-MATNR = LS_S602-MATNR.
```

#### Cú pháp mới (新)

```abap
SELECT SINGLE NAME1
  INTO OS_OUT-AGNAM
  FROM KNA1
  WHERE KUNNR EQ IS_VBRP_VBRK-KUNAG.

OS_OUT-KUNRG = IS_VBRP_VBRK-KUNRG.
OS_OUT-MATNR = LS_S602-MATNR.
```

> `IS_VBRP_VRPMA` (kiểu index) → `IS_VBRP_VBRK` (kiểu bảng gốc), và field `KUNNR` → `KUNRG` theo bảng mapping.
