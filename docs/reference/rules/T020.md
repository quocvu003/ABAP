## T020 - BSIS data được bổ sung từ ACDOCA

* **ID:** `T020`
* **Nội dung lỗi (エラー内容):** Có pattern không lấy được dữ liệu đầy đủ từ `BSIS` sau S/4HANA do dữ liệu FI được tích hợp vào `ACDOCA`.
* **Cách khắc phục (対応方法):** Về cơ bản vẫn lấy dữ liệu từ `BSIS`. Với các pattern không lấy được từ `BSIS`, lấy `BKPF-BSTAT = 'U'`, gọi `FAGL_GET_GL_DOCUMENT` để lấy dữ liệu ACDOCA dạng `BSEG`, sau đó lọc未消込 bằng `AUGBL = SPACE` cùng với điều kiện positive giống SELECT/logic gốc và merge vào internal table/structure kiểu `BSIS`.

### Nguyên tắc xử lý

* Về cơ bản, vẫn giữ logic lấy dữ liệu từ `BSIS`.
* **Điều kiện áp dụng — chỉ thêm fallback khi SELECT gốc CÓ THỂ chứa `BSTAT='U'`:** nếu WHERE gốc đã loại trừ `'U'` thì **giữ nguyên SELECT, KHÔNG thêm fallback**. Các trường hợp loại trừ: (a) lọc `BSTAT` khác `'U'` tường minh; (b) điều kiện mâu thuẫn với bản chất `BSIS` khi kết hợp `AUGBL = SPACE` của rule — ví dụ SELECT `BSIS` bị pin `AUGBL = <khác rỗng>` (BSIS = 未消込, AUGBL luôn SPACE) → mâu thuẫn `AUGBL = SPACE` → fallback luôn 0 dòng → bỏ hẳn; (c) **chứng từ nguồn gốc Logistics** không bao giờ `BSTAT='U'` — nếu luồng xử lý là `BSIS` → `VBRP` (chi tiết hóa đơn SD) → `VBAP` (chi tiết đơn hàng SD), hoặc chứng từ gắn `AWTYP = 'VBRK'`/`'MKPF'`, thì **cả bước đọc `BSIS` lẫn bước đọc `VBRP`/`VBAP` đều KHÔNG cần sửa** (`VBRP`/`VBAP` là bảng SD, ngoài phạm vi migration ACDOCA); (d) `BKPF-GLVOR = 'RMRP'` (Đối chiếu hóa đơn MIRO — Logistics) → không có `BSTAT='U'` → bỏ fallback (ngược lại `GLVOR = 'RFBU'` — bút toán FI tổng quát như khấu hao 減価償却 / đánh giá ngoại tệ 外貨評価 — CÓ THỂ `BSTAT='U'` → **LÀ đối tượng cần sửa 改修候補**); (e) nguồn dữ liệu gốc là `ZPUT0037` (検収明細データ — chi tiết nghiệm thu) → không phát sinh `BSTAT='U'` → không cần sửa.
* Có pattern không lấy được dữ liệu từ `BSIS`; khi đó lấy `BKPF-BSTAT = 'U'`.
* **Buffer nhận `ET_BSEG` khai `TYPE TABLE OF BSEG`** (hoặc `FAGL_T_BSEG` — tương đương). Trên S/4, `BSEG` đã có sẵn header field prefix `H_` (`H_BLART` thay cho `BLART`, `H_HWAER`...). Lấy field header qua `H_<field>` trực tiếp từ buffer BSEG. KHÔNG dùng `TABLE OF BSIS/BSAS` (type conflict).
* Với document status `U`, dùng:
  * `CL_FINS_ACDOC_UTIL=>GET_LEADING_LEDGER( )` để lấy leading ledger.
  * FM `FAGL_GET_GL_DOCUMENT` để lấy dữ liệu ACDOCA dưới dạng `BSEG`.
* Sau khi lấy dữ liệu từ FM, lọc bằng điều kiện positive giống SELECT/logic gốc.
* T020 mặc định phải thêm điều kiện `AUGBL = SPACE` khi lọc `LT_BSEG_ACDOC` để chỉ bổ sung未消込/open item tương ứng `BSIS`.
* Sau đó map dữ liệu sang structure/table kiểu `BSIS` và merge/appended về kết quả `BSIS` cũ.
* Sau khi gọi `FAGL_GET_GL_DOCUMENT`, dùng `LOOP AT LT_BSEG_ACDOC ... WHERE ...` với điều kiện positive giống điều kiện SELECT gốc, không dùng `DELETE ... WHERE` với điều kiện ngược.
* Được dùng `MOVE-CORRESPONDING` cho table/structure merge khi source/target có cấu trúc tương ứng.
* Không thêm pseudo-comment `"#EC CI_DB_OPERATION_OK[2431747]` vào dòng `FROM BSIS`.
* Nếu cùng một FORM xử lý cả `BSAS` và `BSIS`:
  * Nếu BSIS và BSAS map sang **structure đích khác nhau** → **tách dữ liệu bổ sung ACDOCA ra 2 internal table riêng biệt** (ví dụ `LT_BSAS_ACDOC` và `LT_BSIS_ACDOC`), sau đó mới append về kết quả chung. Không dùng chung một internal table trung gian cho cả hai.
  * Nếu BSIS và BSAS cùng map sang **một structure đích giống nhau** (field list y hệt) → được phép gộp xử lý trong **một `LOOP`** và append thẳng về kết quả chung, nhưng điều kiện lọc bắt buộc viết **tường minh từng nhánh** (xem nguyên tắc "code tường minh" bên dưới).
* **Ưu tiên code tường minh** (theo nguyên tắc chung — Section 0 của `sap_abap_error_reference.md`): khi điều kiện lọc gộp nhiều nguồn, viết tách bạch từng nhánh để thấy rõ nhánh nào tương ứng BSIS / BSAS, **không rút gọn** dù kết quả tương đương. Ví dụ ưu tiên:

  ```abap
  IF LS_BSEG_ACDOC-AUGBL = SPACE                                    " BSIS相当: 未消込（オープン）
  OR ( LS_BSEG_ACDOC-AUGBL <> SPACE                                " BSAS相当: 消込済
   AND LS_BSEG_ACDOC-AUGDT > P_BUDAT ).                            "          かつ基準日後に消込
  ```

  Không nên rút gọn thành `IF AUGBL = SPACE OR AUGDT > P_BUDAT.` — tuy đúng kết quả nhưng không thể hiện rõ nhánh BSAS. (Dấu ngoặc `(` `)` phải có khoảng trắng 2 bên.)

### BKPF key mapping

| ECC | S/4 | Nội dung |
| --- | --- | --- |
| `BUKRS` | `RBUKRS` | 会社コード |
| `BELNR` | `BELNR` | 会計伝票の伝票番号 |
| `GJAHR` | `GJAHR` | 会計年度 |

### Test data cho 伝票ステータス 'U' (外貨評価伝票)

* Phía **JAE** đã tạo sẵn **chứng từ đánh giá ngoại tệ (外貨評価伝票)** đối ứng `BSEG`/`BSIS`/`BSAS`; khi tra bảng `BKPF` các chứng từ này có `BSTAT = 'U'` → dùng làm **dữ liệu test**.
* Môi trường: **S/4開発機 CLT:571**. Do khác cấu hình IMG so với ECC開発機, chứng từ đánh giá ngoại tệ **chỉ tạo được với 会社コード (`BUKRS`) = `'JA00'`**.
* **伝票番号 (số chứng từ):** `100000020`, `100000021`, `100000023`.
* Là chứng từ **変更不可 (không thể thay đổi)** → dùng tự do cho test **chức năng 照会 (hiển thị/truy vấn)**.

### Structure retrieval

#### Cú pháp cũ (旧)

```abap
SELECT ZUONR
  FROM BSIS
  INTO LV_ZUONR
  WHERE BUKRS = P_BUKRS
    AND GJAHR = LS_ZFIT0095A-RYEAR
    AND BELNR = LS_ZFIT0095A-BELNR
    AND BUZEI = LS_ZFIT0095A-ZZBUZEI_FI_SL.
  EXIT.
ENDSELECT.
```

#### Cú pháp mới (新)

```abap
SELECT ZUONR
  FROM BSIS
  INTO LV_ZUONR
  WHERE BUKRS = P_BUKRS
    AND GJAHR = LS_ZFIT0095A-RYEAR
    AND BELNR = LS_ZFIT0095A-BELNR
    AND BUZEI = LS_ZFIT0095A-ZZBUZEI_FI_SL.
  EXIT.
ENDSELECT.

IF SY-SUBRC <> 0.
  SELECT SINGLE *
    FROM BKPF
    INTO OS_BKPF
    WHERE BUKRS EQ P_BUKRS
      AND BELNR EQ LS_ZFIT0095A-BELNR
      AND GJAHR EQ LS_ZFIT0095A-RYEAR.

  IF OS_BKPF-BSTAT = 'U'.
    LEADING_LEDGER = CL_FINS_ACDOC_UTIL=>GET_LEADING_LEDGER( ).

    CALL FUNCTION 'FAGL_GET_GL_DOCUMENT'
      EXPORTING
        I_RLDNR           = LEADING_LEDGER
        I_BUKRS           = OS_BKPF-BUKRS
        I_BELNR           = OS_BKPF-BELNR
        I_GJAHR           = OS_BKPF-GJAHR
        I_BSTAT           = OS_BKPF-BSTAT
      IMPORTING
        ET_BSEG           = IT_BSEG[]
      EXCEPTIONS
        NOT_FOUND         = 1
        OTHERS            = 2.

    LOOP AT IT_BSEG[] ASSIGNING <LS_BSEG>
      WHERE AUGBL = ''
        AND BUZEI = LS_ZFIT0095A-ZZBUZEI_FI_SL.
      LV_ZUONR = <LS_BSEG>-ZUONR.
      EXIT.
    ENDLOOP.
  ENDIF.
ENDIF.
```

> Nếu điều kiện lấy `BSIS` đã có sẵn thông tin `BKPF`, không cần `SELECT BKPF` lại; dùng structure `BKPF` đã取得済み.

### Table retrieval

#### Cú pháp cũ — table (旧)

```abap
SELECT HKONT
       GJAHR
       BELNR
       BUZEI
       WAERS
  FROM BSIS
  INTO TABLE LT_BSIS
  FOR ALL ENTRIES IN IT_BELNR
  WHERE BUKRS = P_BUKRS
    AND HKONT LIKE '210399%'
    AND GJAHR = IT_BELNR-GJAHR
    AND BELNR = IT_BELNR-BELNR.
```

#### Cú pháp mới — table (新)

```abap
SELECT HKONT
       GJAHR
       BELNR
       BUZEI
       WAERS
  FROM BSIS
  INTO TABLE LT_BSIS
  FOR ALL ENTRIES IN IT_BELNR
  WHERE BUKRS = P_BUKRS
    AND HKONT LIKE '210399%'
    AND GJAHR = IT_BELNR-GJAHR
    AND BELNR = IT_BELNR-BELNR.

SELECT *
  INTO TABLE OT_BKPF
  FROM BKPF
  FOR ALL ENTRIES IN IT_BELNR
  WHERE BUKRS = P_BUKRS
    AND GJAHR = IT_BELNR-GJAHR
    AND BELNR = IT_BELNR-BELNR.

LOOP AT OT_BKPF ASSIGNING <LS_BKPF>
  WHERE BSTAT = 'U'.

  LEADING_LEDGER = CL_FINS_ACDOC_UTIL=>GET_LEADING_LEDGER( ).

  CALL FUNCTION 'FAGL_GET_GL_DOCUMENT'
    EXPORTING
      I_RLDNR           = LEADING_LEDGER
      I_BUKRS           = <LS_BKPF>-BUKRS
      I_BELNR           = <LS_BKPF>-BELNR
      I_GJAHR           = <LS_BKPF>-GJAHR
      I_BSTAT           = <LS_BKPF>-BSTAT
    IMPORTING
      ET_BSEG           = IT_BSEG[]
    EXCEPTIONS
      NOT_FOUND         = 1
      OTHERS            = 2.

  LOOP AT IT_BSEG[] ASSIGNING <LS_BSEG>
    WHERE AUGBL = ''
      AND HKONT LIKE '210399%'.
    MOVE-CORRESPONDING <LS_BSEG> TO LS_BSIS.
    APPEND LS_BSIS TO LT_BSIS.
  ENDLOOP.
ENDLOOP.
```
