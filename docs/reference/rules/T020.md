## T020 - BSIS data được bổ sung từ ACDOCA

* **ID:** `T020`
* **Nội dung lỗi (エラー内容):** Có pattern không lấy được dữ liệu đầy đủ từ `BSIS` sau S/4HANA do dữ liệu FI được tích hợp vào `ACDOCA`.
* **Cách khắc phục (対応方法):** Về cơ bản vẫn lấy dữ liệu từ `BSIS`. Với các pattern không lấy được từ `BSIS`, lấy `BKPF-BSTAT = 'U'`, gọi `FAGL_GET_GL_DOCUMENT` để lấy dữ liệu ACDOCA dạng `BSEG`, sau đó lọc未消込 bằng `AUGBL = SPACE` cùng với điều kiện positive giống SELECT/logic gốc và merge vào internal table/structure kiểu `BSIS`.

### Nguyên tắc xử lý

* Về cơ bản, vẫn giữ logic lấy dữ liệu từ `BSIS`.
* Có pattern không lấy được dữ liệu từ `BSIS`; khi đó lấy `BKPF-BSTAT = 'U'`.
* Với document status `U`, dùng:
  * `CL_FINS_ACDOC_UTIL=>GET_LEADING_LEDGER( )` để lấy leading ledger.
  * FM `FAGL_GET_GL_DOCUMENT` để lấy dữ liệu ACDOCA dưới dạng `BSEG`.
* Sau khi lấy dữ liệu từ FM, lọc bằng điều kiện positive giống SELECT/logic gốc.
* T020 mặc định phải thêm điều kiện `AUGBL = SPACE` khi lọc `LT_BSEG_ACDOC` để chỉ bổ sung未消込/open item tương ứng `BSIS`.
* Sau đó map dữ liệu sang structure/table kiểu `BSIS` và merge/appended về kết quả `BSIS` cũ.
* Sau khi gọi `FAGL_GET_GL_DOCUMENT`, dùng `LOOP AT LT_BSEG_ACDOC ... WHERE ...` với điều kiện positive giống điều kiện SELECT gốc, không dùng `DELETE ... WHERE` với điều kiện ngược.
* Được dùng `MOVE-CORRESPONDING` cho table/structure merge khi source/target có cấu trúc tương ứng.
* Không thêm pseudo-comment `"#EC CI_DB_OPERATION_OK[2431747]` vào dòng `FROM BSIS`.
* Nếu cùng một FORM xử lý cả `BSAS` và `BSIS`, **tách dữ liệu bổ sung ACDOCA ra 2 internal table riêng biệt** (ví dụ `LT_BSAS_ACDOC` và `LT_BSIS_ACDOC`), sau đó mới append về kết quả chung. Không dùng chung một internal table trung gian cho cả hai.

### BKPF key mapping

| ECC | S/4 | Nội dung |
| --- | --- | --- |
| `BUKRS` | `RBUKRS` | 会社コード |
| `BELNR` | `BELNR` | 会計伝票の伝票番号 |
| `GJAHR` | `GJAHR` | 会計年度 |

### Structure retrieval

#### Cú pháp cũ (旧)

```abap
SELECT ZUONR
  FROM BSIS
  INTO LV_ZUONR
  WHERE BUKRS = P_BUKRS
    AND GJAHR = LS_ZFIT0095A-RYEAR
    AND BELNR = LS_ZFIT0095A-BELNR
    AND BUZEI = LS_ZFIT0095A-ZZBUZEI_FI_SL.
  EXIT.
ENDSELECT.
```

#### Cú pháp mới (新)

```abap
SELECT ZUONR
  FROM BSIS
  INTO LV_ZUONR
  WHERE BUKRS = P_BUKRS
    AND GJAHR = LS_ZFIT0095A-RYEAR
    AND BELNR = LS_ZFIT0095A-BELNR
    AND BUZEI = LS_ZFIT0095A-ZZBUZEI_FI_SL.
  EXIT.
ENDSELECT.

IF SY-SUBRC <> 0.
  SELECT SINGLE *
    FROM BKPF
    INTO OS_BKPF
    WHERE BUKRS EQ P_BUKRS
      AND BELNR EQ LS_ZFIT0095A-BELNR
      AND GJAHR EQ LS_ZFIT0095A-RYEAR.

  IF OS_BKPF-BSTAT = 'U'.
    LEADING_LEDGER = CL_FINS_ACDOC_UTIL=>GET_LEADING_LEDGER( ).

    CALL FUNCTION 'FAGL_GET_GL_DOCUMENT'
      EXPORTING
        I_RLDNR           = LEADING_LEDGER
        I_BUKRS           = OS_BKPF-BUKRS
        I_BELNR           = OS_BKPF-BELNR
        I_GJAHR           = OS_BKPF-GJAHR
        I_BSTAT           = OS_BKPF-BSTAT
      IMPORTING
        ET_BSEG           = IT_BSEG[]
      EXCEPTIONS
        NOT_FOUND         = 1
        OTHERS            = 2.

    LOOP AT IT_BSEG[] ASSIGNING <LS_BSEG>
      WHERE AUGBL = ''
        AND BUZEI = LS_ZFIT0095A-ZZBUZEI_FI_SL.
      LV_ZUONR = <LS_BSEG>-ZUONR.
      EXIT.
    ENDLOOP.
  ENDIF.
ENDIF.
```

> Nếu điều kiện lấy `BSIS` đã có sẵn thông tin `BKPF`, không cần `SELECT BKPF` lại; dùng structure `BKPF` đã取得済み.

### Table retrieval

#### Cú pháp cũ — table (旧)

```abap
SELECT HKONT
       GJAHR
       BELNR
       BUZEI
       WAERS
  FROM BSIS
  INTO TABLE LT_BSIS
  FOR ALL ENTRIES IN IT_BELNR
  WHERE BUKRS = P_BUKRS
    AND HKONT LIKE '210399%'
    AND GJAHR = IT_BELNR-GJAHR
    AND BELNR = IT_BELNR-BELNR.
```

#### Cú pháp mới — table (新)

```abap
SELECT HKONT
       GJAHR
       BELNR
       BUZEI
       WAERS
  FROM BSIS
  INTO TABLE LT_BSIS
  FOR ALL ENTRIES IN IT_BELNR
  WHERE BUKRS = P_BUKRS
    AND HKONT LIKE '210399%'
    AND GJAHR = IT_BELNR-GJAHR
    AND BELNR = IT_BELNR-BELNR.

SELECT *
  INTO TABLE OT_BKPF
  FROM BKPF
  FOR ALL ENTRIES IN IT_BELNR
  WHERE BUKRS = P_BUKRS
    AND GJAHR = IT_BELNR-GJAHR
    AND BELNR = IT_BELNR-BELNR.

LOOP AT OT_BKPF ASSIGNING <LS_BKPF>
  WHERE BSTAT = 'U'.

  LEADING_LEDGER = CL_FINS_ACDOC_UTIL=>GET_LEADING_LEDGER( ).

  CALL FUNCTION 'FAGL_GET_GL_DOCUMENT'
    EXPORTING
      I_RLDNR           = LEADING_LEDGER
      I_BUKRS           = <LS_BKPF>-BUKRS
      I_BELNR           = <LS_BKPF>-BELNR
      I_GJAHR           = <LS_BKPF>-GJAHR
      I_BSTAT           = <LS_BKPF>-BSTAT
    IMPORTING
      ET_BSEG           = IT_BSEG[]
    EXCEPTIONS
      NOT_FOUND         = 1
      OTHERS            = 2.

  LOOP AT IT_BSEG[] ASSIGNING <LS_BSEG>
    WHERE AUGBL = ''
      AND HKONT LIKE '210399%'.
    MOVE-CORRESPONDING <LS_BSEG> TO LS_BSIS.
    APPEND LS_BSIS TO LT_BSIS.
  ENDLOOP.
ENDLOOP.
```
