## 1.1. B002 - Lỗi Batch Input hủy đăng ký nhập xuất kho (MBST)

* **ID:** `B002`
* **Nội dung lỗi (エラー内容):** Lệnh gọi Batch Input cho giao dịch Hủy chứng từ nhập xuất kho (`MBST`) không còn được hỗ trợ trên S/4HANA.
* **Cách khắc phục (対応方法):** Thay thế bằng BAPI `BAPI_GOODSMVT_CANCEL`.

### Cú pháp cũ (旧)

```abap
CALL TRANSACTION 'MBST' USING TB_BDC
                        MODE 'S'
                        UPDATE 'N'
                        MESSAGES INTO TB_MESSAGE.
```

### Cú pháp mới (新)

```abap
CALL FUNCTION 'BAPI_GOODSMVT_CANCEL'
  EXPORTING
    MATERIALDOCUMENT      = '1234567'       " Số chứng từ (MBLNR)
    MATDOCUMENTYEAR       = '2020'          " Năm chứng từ (MJAHR)
    GOODSMVT_PSTING_DATE  = '20200612'      " Ngày hạch toán (BUDAT)
  IMPORTING
    GOODSMVT_HEADER       = LTH_HEADER
  TABLES
    GOODSMVT_MATDOCITEM   = LTD_MATDOCITEM
    RETURN                = LTD_RETURN.

* Kiểm tra lỗi (エラー判定)
LOOP AT LTD_RETURN INTO LTH_RETURN WHERE TYPE = 'E' OR TYPE = 'A' OR TYPE = 'X'.
  EXIT.
ENDLOOP.

IF SY-SUBRC = 0.
  * Thất bại -> BAPI Rollback (BAPIロールバック)
  CALL FUNCTION 'BAPI_TRANSACTION_ROLLBACK'.
  * Thực hiện xử lý lỗi và xuất message lỗi (Phụ thuộc vào logic cũ / エラー時の処理、メッセージの出力が必要。既存ロジックに依存。)
ELSE.
  * Thành công -> BAPI Commit (BAPIコミット)
  CALL FUNCTION 'BAPI_TRANSACTION_COMMIT'
    EXPORTING
      WAIT = 'X'.
  * Thực hiện xử lý thành công và xuất message thành công (Phụ thuộc vào logic cũ / 正常時の処理、メッセージの出力が必要。既存ロジックに依存。)
  * LƯU Ý QUAN TRỌNG: Khác với BDC (Batch Input), BAPI không tự động hiển thị message thành công. Do đó, bắt buộc phải viết code để xuất thông báo thành công (ví dụ hiển thị số chứng từ vật tư LTH_HEADER-MAT_DOC vừa tạo) nếu thiết kế cũ không chỉ định.
  * (※.正常終了で指定がない場合は、必ずメッセージを出力する。BAPIの場合、自動でメッセージ出力しないため)
  MESSAGE S060(M7) WITH LTH_HEADER-MAT_DOC.
ENDIF.
```
