## T021 - BSAS data được bổ sung từ ACDOCA

* **ID:** `T021`
* **Nội dung lỗi (エラー内容):** Có pattern không lấy được dữ liệu đầy đủ từ `BSAS` sau S/4HANA do dữ liệu FI được tích hợp vào `ACDOCA`.
* **Cách khắc phục (対応方法):** Về cơ bản vẫn lấy dữ liệu từ `BSAS`. Với các pattern không lấy được từ `BSAS`, lấy `BKPF-BSTAT = 'U'`, gọi `FAGL_GET_GL_DOCUMENT` để lấy dữ liệu ACDOCA dạng `BSEG`, sau đó lọc消込済 bằng `AUGBL <> SPACE` cùng với điều kiện positive giống SELECT/logic gốc và merge vào internal table/structure kiểu `BSAS`.

### Nguyên tắc xử lý

* Về cơ bản, vẫn giữ logic lấy dữ liệu từ `BSAS`.
* **Điều kiện áp dụng — chỉ thêm fallback khi SELECT gốc CÓ THỂ chứa `BSTAT='U'`:** nếu WHERE gốc đã loại trừ `'U'` thì **giữ nguyên SELECT, KHÔNG thêm fallback**. Các trường hợp loại trừ: (a) lọc `BSTAT` khác `'U'` tường minh; (b) điều kiện mâu thuẫn với bản chất `BSAS` khi kết hợp `AUGBL <> SPACE` của rule — ví dụ SELECT `BSAS` bị pin `AUGBL = SPACE` (BSAS = 消込済) → mâu thuẫn `AUGBL <> SPACE` → fallback luôn 0 dòng → bỏ hẳn.
* Có pattern không lấy được dữ liệu từ `BSAS`; khi đó lấy `BKPF-BSTAT = 'U'`.
* **Buffer nhận `ET_BSEG` khai `TYPE TABLE OF BSEG`** (hoặc `FAGL_T_BSEG` — tương đương). Trên S/4, `BSEG` đã có sẵn header field prefix `H_` (`H_BLART` thay cho `BLART`, `H_HWAER`...). Lấy field header qua `H_<field>` trực tiếp từ buffer BSEG. KHÔNG dùng `TABLE OF BSIS/BSAS` (type conflict).
* Với document status `U`, dùng:
  * `CL_FINS_ACDOC_UTIL=>GET_LEADING_LEDGER( )` để lấy leading ledger.
  * FM `FAGL_GET_GL_DOCUMENT` để lấy dữ liệu ACDOCA dưới dạng `BSEG`.
* Sau khi lấy dữ liệu từ FM, lọc bằng điều kiện positive giống SELECT/logic gốc.
* T021 mặc định phải thêm điều kiện `AUGBL <> SPACE` khi lọc `LT_BSEG_ACDOC` để chỉ bổ sung消込済/cleared item tương ứng `BSAS`.
* Sau đó map dữ liệu sang structure/table kiểu `BSAS` và merge/appended về kết quả `BSAS` cũ.
* Sau khi gọi `FAGL_GET_GL_DOCUMENT`, dùng `LOOP AT LT_BSEG_ACDOC ... WHERE ...` với điều kiện positive giống điều kiện SELECT gốc, không dùng `DELETE ... WHERE` với điều kiện ngược.
* Được dùng `MOVE-CORRESPONDING` cho table/structure merge khi source/target có cấu trúc tương ứng.
* Không thêm pseudo-comment `"#EC CI_DB_OPERATION_OK[2431747]` vào dòng `FROM BSAS`.
* **Ưu tiên code tường minh** (theo nguyên tắc chung — Section 0 của `sap_abap_error_reference.md`): khi lọc `LT_BSEG_ACDOC`, viết điều kiện tách bạch (vd `AUGBL <> SPACE AND AUGDT > P_BUDAT`) để thể hiện rõ "消込済 + clear sau key date", không rút gọn dựa vào giá trị mặc định của field. Nếu cùng FORM gộp cả BSIS + BSAS trong một `LOOP`, xem ví dụ điều kiện tường minh ở [`T020`](T020.md).

### BKPF key mapping

| ECC | S/4 | Nội dung |
| --- | --- | --- |
| `BUKRS` | `RBUKRS` | 会社コード |
| `BELNR` | `BELNR` | 会計伝票の伝票番号 |
| `GJAHR` | `GJAHR` | 会計年度 |

### Structure retrieval

#### Cú pháp cũ (旧)

```abap
SELECT SINGLE WERKS
  FROM BSAS
  INTO LS_FI-ZZWERKS_FR
  WHERE BUKRS = P_BUKRS
    AND BELNR = LS_FI-BELNR
    AND GJAHR = LS_FIT95A-RYEAR
    AND BUZEI = LS_FI-ZZBUZEI_FI_SL.
```

#### Cú pháp mới (新)

```abap
SELECT SINGLE WERKS
  FROM BSAS
  INTO LS_FI-ZZWERKS_FR
  WHERE BUKRS = P_BUKRS
    AND BELNR = LS_FI-BELNR
    AND GJAHR = LS_FIT95A-RYEAR
    AND BUZEI = LS_FI-ZZBUZEI_FI_SL.

IF SY-SUBRC <> 0.
  SELECT SINGLE *
    FROM BKPF
    INTO OS_BKPF
    WHERE BUKRS EQ P_BUKRS
      AND BELNR EQ LS_FI-BELNR
      AND GJAHR EQ LS_FIT95A-RYEAR.

  IF OS_BKPF-BSTAT = 'U'.
    LEADING_LEDGER = CL_FINS_ACDOC_UTIL=>GET_LEADING_LEDGER( ).

    CALL FUNCTION 'FAGL_GET_GL_DOCUMENT'
      EXPORTING
        I_RLDNR           = LEADING_LEDGER
        I_BUKRS           = OS_BKPF-BUKRS
        I_BELNR           = OS_BKPF-BELNR
        I_GJAHR           = OS_BKPF-GJAHR
        I_BSTAT           = OS_BKPF-BSTAT
      IMPORTING
        ET_BSEG           = IT_BSEG[]
      EXCEPTIONS
        NOT_FOUND         = 1
        OTHERS            = 2.

    LOOP AT IT_BSEG[] ASSIGNING <LS_BSEG>
      WHERE AUGBL <> ''
        AND BUZEI = LS_FI-ZZBUZEI_FI_SL.
      LS_FI-ZZWERKS_FR = <LS_BSEG>-WERKS.
      EXIT.
    ENDLOOP.
  ENDIF.
ENDIF.
```

> Nếu điều kiện lấy `BSAS` đã có sẵn thông tin `BKPF`, không cần `SELECT BKPF` lại; dùng structure `BKPF` đã取得済み.

### Table retrieval

#### Cú pháp cũ — table (旧)

```abap
SELECT BUKRS
       GJAHR
       BELNR
       AUGBL
       XRAGL
  FROM BSAS
  APPENDING TABLE GT_BSAS_TBL
  FOR ALL ENTRIES IN LT_REGUP_TBL
  WHERE BUKRS = LS_REGUH-ZBUKR
    AND GJAHR = LV_YEAR
    AND BELNR = LT_REGUP_TBL-VBLNR.
```

#### Cú pháp mới — table (新)

```abap
SELECT BUKRS
       GJAHR
       BELNR
       AUGBL
       XRAGL
  FROM BSAS
  APPENDING TABLE GT_BSAS_TBL
  FOR ALL ENTRIES IN LT_REGUP_TBL
  WHERE BUKRS = LS_REGUH-ZBUKR
    AND GJAHR = LV_YEAR
    AND BELNR = LT_REGUP_TBL-VBLNR.

SELECT *
  INTO TABLE OT_BKPF
  FROM BKPF
  FOR ALL ENTRIES IN LT_REGUP_TBL
  WHERE BUKRS = LS_REGUH-ZBUKR
    AND GJAHR = LV_YEAR
    AND BELNR = LT_REGUP_TBL-VBLNR.

LOOP AT OT_BKPF ASSIGNING <LS_BKPF>
  WHERE BSTAT = 'U'.

  LEADING_LEDGER = CL_FINS_ACDOC_UTIL=>GET_LEADING_LEDGER( ).

  CALL FUNCTION 'FAGL_GET_GL_DOCUMENT'
    EXPORTING
      I_RLDNR           = LEADING_LEDGER
      I_BUKRS           = <LS_BKPF>-BUKRS
      I_BELNR           = <LS_BKPF>-BELNR
      I_GJAHR           = <LS_BKPF>-GJAHR
      I_BSTAT           = <LS_BKPF>-BSTAT
    IMPORTING
      ET_BSEG           = IT_BSEG[]
    EXCEPTIONS
      NOT_FOUND         = 1
      OTHERS            = 2.

  LOOP AT IT_BSEG[] ASSIGNING <LS_BSEG>
    WHERE AUGBL <> ''.
    MOVE-CORRESPONDING <LS_BSEG> TO GS_BSAS_TBL.
    APPEND GS_BSAS_TBL TO GT_BSAS_TBL.
  ENDLOOP.
ENDLOOP.
```
