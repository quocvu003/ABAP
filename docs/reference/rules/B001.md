## 1. B001 - Lỗi Batch Input đăng ký Nhập/Xuất kho (MBXX)

* **ID:** `B001`
* **Nội dung lỗi (エラー内容):** Không được phép sử dụng Batch Input cho các giao dịch đăng ký nhập xuất kho (MBXX) trên S/4HANA.
* **Cách khắc phục (対応方法):** Thay thế bằng BAPI `BAPI_GOODSMVT_CREATE`.

### Cú pháp cũ (旧)

```abap
CALL TRANSACTION 'MB11' USING TB_BDC
                        MODE 'S'
                        UPDATE 'N'
                        MESSAGES INTO TB_MESSAGE.
```

### Cú pháp mới (新)

```abap
* BAPI_GOODSMVT_CREATE Parameter Mapping
CALL FUNCTION 'BAPI_GOODSMVT_CREATE'
  EXPORTING
    GOODSMVT_HEADER = TH_HEADER
    GOODSMVT_CODE   = TH_CODE
  IMPORTING
    GOODSMVT        = TH_OUT
  TABLES
    GOODSMVT_ITEM   = TD_ITEM
    RETURN          = TD_RETURN.
```

#### Bảng tra cứu mã nghiệp vụ `GOODSMVT_CODE`

* `01`: Nhập kho cho Đơn mua hàng (GR for Purchase Order / 購買発注に対する入庫)
* `02`: Nhập kho cho Lệnh sản xuất (GR for Production Order / 製造計画に対する入庫)
* `03`: Xuất kho (GI - Goods Issue / 出庫)
* `04`: Chuyển kho (Transfer Posting / 在庫転送)
* `05`: Nhập kho khác (Other Goods Receipts / その他入庫)
* `06`: Hủy di chuyển kho (Cancellation of Goods Movement / 在庫移動の取消)

#### Mẫu khai báo & Xử lý chi tiết (Sample Source)

```abap
* ローカル定義（BAPI用） - Khai báo biến cục bộ cho BAPI
DATA: LTH_HEADER  TYPE BAPI2017_GM_HEAD_01,              "入出庫伝票ヘッダ (Header)
      LTH_CODE    TYPE BAPI2017_GM_CODE.                 "トランザクション (GM Code)
DATA: LTD_ITEM    TYPE TABLE OF BAPI2017_GM_ITEM_CREATE, "入出庫伝票明細 (Items Table)
      LTH_ITEM    LIKE LINE OF  LTD_ITEM.                "入出庫伝票明細 (Item Row)
DATA: LTH_HEADRET TYPE BAPI2017_GM_HEAD_RET,             "入出庫伝票結果 (Result Header)
      LTD_RETURN  TYPE TABLE OF BAPIRET2,                "リターンMSG (Return Messages Table)
      LTH_RETURN  LIKE LINE OF  LTD_RETURN.              "リターンMSG (Return Message Row)

CLEAR: LTH_HEADER, LTH_CODE, LTH_ITEM, LTH_HEADRET, LTH_RETURN.
REFRESH: LTD_ITEM, LTD_RETURN.

* GM Code Assignment
LTH_CODE-GM_CODE = '01'.

* Header Mapping
LTH_HEADER-DOC_DATE   = PI_KDATE.    "伝票日付 (Posting Date)
LTH_HEADER-PSTNG_DATE = PI_PI_KDATE. "転記日付 (Document Date)
LTH_HEADER-HEADER_TXT = PI_EBELN.    "ヘッダText (Header Text)

* Item Loop & Mapping
LOOP AT TD_XXXX INTO TH_XXXX.
  CLEAR LTH_ITEM.
  
  * Convert Quantity (Sử dụng WRITE để format số lượng)
  WRITE TH_XXXX-NMENGE TO L_MENGE UNIT XXXX-BPRME LEFT-JUSTIFIED NO-GROUPING NO-SIGN.
  
  LTH_ITEM-MOVE_TYPE = '101'.          "移動タイプ (Movement Type)
  LTH_ITEM-PO_NUMBER = TH_XXXX-EBELN.  "発注伝票 (PO Number)
  LTH_ITEM-PO_ITEM   = TH_XXXX-EBELP.  "発注明細 (PO Item)
  LTH_ITEM-ENTRY_QNT = L_MENGE.        "発注数量 (Quantity)
  LTH_ITEM-ENTRY_UOM = TH_XXXX-BPRME.  "数量単位 (UoM)
  LTH_ITEM-ITEM_TEXT = TH_XXXX-SGTEXT. "テキスト (Text)
  LTH_ITEM-MVT_IND   = 'B'.            "移動区分 (Movement Indicator)
  
  APPEND LTH_ITEM TO LTD_ITEM.
ENDLOOP.

* Thực thi BAPI (BAPIの実行)
CALL FUNCTION 'BAPI_GOODSMVT_CREATE'
  EXPORTING
    GOODSMVT_HEADER = LTH_HEADER
    GOODSMVT_CODE   = LTH_CODE
  IMPORTING
    GOODSMVT_HEADRET = LTH_HEADRET
  TABLES
    GOODSMVT_ITEM   = LTD_ITEM
    RETURN          = LTD_RETURN.

* Kiểm tra lỗi (エラー判定)
LOOP AT LTD_RETURN INTO LTH_RETURN WHERE TYPE = 'E' OR TYPE = 'A' OR TYPE = 'X'.
  EXIT.
ENDLOOP.

IF SY-SUBRC = 0.
  * Lỗi -> Rollback (BAPIロールバック)
  CALL FUNCTION 'BAPI_TRANSACTION_ROLLBACK'.
  * Thực hiện xử lý lỗi và xuất message lỗi (Phụ thuộc vào logic thông báo lỗi cũ của chương trình)
ELSE.
  * Thành công -> Commit (BAPIコミット)
  CALL FUNCTION 'BAPI_TRANSACTION_COMMIT'
    EXPORTING
      WAIT = 'X'.
  
  * Thực hiện xử lý thành công và xuất message thành công (Phụ thuộc vào logic cũ)
  * LƯU Ý QUAN TRỌNG: Khác với BDC (Batch Input), BAPI không tự động hiển thị message thành công. Do đó, bắt buộc phải viết code để xuất thông báo thành công (ví dụ hiển thị số chứng từ vật tư LTH_HEADRET-MAT_DOC vừa tạo).
  MESSAGE S060(M7) WITH LTH_HEADRET-MAT_DOC.
ENDIF.
```
