## T019 - FI document table `BSEG` + `BKPF` được tích hợp sang `ACDOCA`

* **ID:** `T019`
* **Nội dung lỗi (エラー内容):** FI伝票の中心テーブルが `BSEG` + `BKPF` から `ACDOCA` に統合される.
* **Cách khắc phục (対応方法):** Vì nhiều field chính của `BSEG` được chuyển sang `ACDOCA`, xử lý lấy `BSEG` cần bổ sung logic lấy dữ liệu từ `ACDOCA` thông qua `FAGL_GET_GL_DOCUMENT` cho các pattern không lấy được đầy đủ từ `BSEG`.

### Nguyên tắc xử lý

* Về cơ bản, vẫn giữ logic lấy dữ liệu từ `BSEG`.
* **Điều kiện áp dụng — chỉ thêm fallback khi SELECT gốc CÓ THỂ chứa `BSTAT='U'`:** nếu WHERE của SELECT gốc đã loại trừ `'U'` (ví dụ lọc `BSTAT` khác `'U'` như `BSTAT = 'A'`) thì SELECT gốc vốn không lấy `'U'` → **giữ nguyên SELECT, KHÔNG thêm fallback** (vẫn giữ pseudo-comment cho `FROM BSEG` để tắt cảnh báo ATC direct-access).
* **Loại trừ theo NGUỒN GỐC chứng từ (`AWTYP`):** nếu WHERE của BKPF (hoặc điều kiện lọc chứng từ) pin `AWTYP = 'VBRK'` (chứng từ hóa đơn SD) hoặc `AWTYP = 'MKPF'` (chứng từ vật tư MM — nhập/xuất kho) thì đây là chứng từ kế toán **sinh ra từ nghiệp vụ Logistics** — luôn ghi đầy đủ `BSEG`, `BSTAT = ' '` (normal), **KHÔNG BAO GIỜ** `BSTAT='U'` (ACDOCA専用). → **giữ nguyên SELECT, KHÔNG thêm fallback** (vẫn giữ pseudo-comment cho `FROM BSEG`). Lý do: nhóm chứng từ Logistics không nằm trong đối tượng ACDOCA-only nên `BSEG` cổ điển đã lấy đủ.
* **Phân loại theo `BKPF-GLVOR` (業務取引 — business transaction):**
  * `GLVOR = 'RMRP'` → chứng từ sinh từ **Đối chiếu hóa đơn (MIRO / MM Invoice Verification)** — nghiệp vụ Logistics → **không có `BSTAT='U'`** → **KHÔNG cần sửa**.
  * `GLVOR = 'RFBU'` → bút toán FI tổng quát, **có khả năng** là chứng từ **Khấu hao (減価償却)** hoặc **Đánh giá ngoại tệ (外貨評価)** → **CÓ THỂ `BSTAT='U'`** → **LÀ đối tượng cần sửa (改修候補)** → thêm fallback bình thường.
* **Loại trừ theo bảng nguồn:** nếu dữ liệu gốc lấy từ `ZPUT0037` (検収明細データ — chi tiết nghiệm thu) thì **KHÔNG cần sửa** (không phát sinh chứng từ `BSTAT='U'`).
* Có pattern không lấy được dữ liệu từ `BSEG`; khi đó lấy `BKPF-BSTAT = 'U'`.
* **Buffer nhận `ET_BSEG` khai `TYPE TABLE OF BSEG`** (hoặc `FAGL_T_BSEG` — tương đương, bản chất là table of BSEG). Trên S/4, structure `BSEG` đã có sẵn header field prefix `H_` (`H_BLART` thay cho `BLART`, `H_HWAER`, `H_BUDAT`...). Lấy field header qua `H_<field>` trực tiếp từ buffer BSEG. KHÔNG dùng `TABLE OF BSIS/BSAS` (type conflict).
* Với document status `U`, dùng:
  * `CL_FINS_ACDOC_UTIL=>GET_LEADING_LEDGER( )` để lấy leading ledger.
  * FM `FAGL_GET_GL_DOCUMENT` để lấy dữ liệu ACDOCA dưới dạng `BSEG`.
* Nếu xử lý dạng table, sau khi lấy dữ liệu từ FM thì merge/appended về internal table `BSEG` cũ.
* Pattern approved cho table merge:
  * Lấy `BKPF-BSTAT = 'U'` vào internal table `LT_BKPF`.
  * Dùng `LOOP AT LT_BKPF INTO DATA(LS_BKPF)`.
  * Sau khi gọi `FAGL_GET_GL_DOCUMENT`, dùng `LOOP AT LT_BSEG_ACDOC ... WHERE ...` với điều kiện positive giống điều kiện SELECT gốc.
  * Không dùng `DELETE ... WHERE` với điều kiện ngược để lọc dữ liệu ACDOCA.
  * Dùng `MOVE-CORRESPONDING` cho table/structure merge khi source/target có cấu trúc tương ứng.
  * Khi thêm pseudo-comment `"#EC CI_DB_OPERATION_OK[2431747]` vào dòng `FROM BSEG`, phải bọc **chỉ dòng `FROM BSEG`** bằng `S/4HANA Migration UPD START/END` — không bọc toàn bộ SELECT statement. Pattern bắt buộc:

  ```abap
  * S/4HANA Migration UPD START
  *   FROM
  *     BSEG
      FROM
        BSEG  "#EC CI_DB_OPERATION_OK[2431747]
  * S/4HANA Migration UPD END
  ```

### BKPF key mapping

| ECC | S/4 | Nội dung |
| --- | --- | --- |
| `BUKRS` | `RBUKRS` | 会社コード |
| `BELNR` | `BELNR` | 会計伝票の伝票番号 |
| `GJAHR` | `GJAHR` | 会計年度 |

### Test data cho 伝票ステータス 'U' (外貨評価伝票)

* Phía **JAE** đã tạo sẵn **chứng từ đánh giá ngoại tệ (外貨評価伝票)** đối ứng `BSEG`/`BSIS`/`BSAS`; khi tra bảng `BKPF` các chứng từ này có `BSTAT = 'U'` → dùng làm **dữ liệu test**.
* Môi trường: **S/4開発機 CLT:571**. Do khác cấu hình IMG so với ECC開発機, chứng từ đánh giá ngoại tệ **chỉ tạo được với 会社コード (`BUKRS`) = `'JA00'`**.
* **伝票番号 (số chứng từ):** `100000020`, `100000021`, `100000023`.
* Là chứng từ **変更不可 (không thể thay đổi)** → dùng tự do cho test **chức năng 照会 (hiển thị/truy vấn)**.

### Structure retrieval

#### Cú pháp cũ (旧)

```abap
SELECT SINGLE *
  FROM BSEG
  INTO OS_BSEG
  WHERE BUKRS EQ P_SALID
    AND BELNR EQ IV_BELNR
    AND GJAHR EQ IV_GJAHR
    AND BUZEI EQ IV_BUZEI.
```

#### Cú pháp mới (新)

```abap
SELECT SINGLE *
  FROM BSEG
  INTO OS_BSEG
  WHERE BUKRS EQ P_SALID
    AND BELNR EQ IV_BELNR
    AND GJAHR EQ IV_GJAHR
    AND BUZEI EQ IV_BUZEI.

IF SY-SUBRC <> 0.
  SELECT SINGLE *
    FROM BKPF
    INTO OS_BKPF
    WHERE BUKRS EQ P_SALID
      AND BELNR EQ IV_BELNR
      AND GJAHR EQ IV_GJAHR.

  IF OS_BKPF-BSTAT = 'U'.
    LEADING_LEDGER = CL_FINS_ACDOC_UTIL=>GET_LEADING_LEDGER( ).

    CALL FUNCTION 'FAGL_GET_GL_DOCUMENT'
      EXPORTING
        I_RLDNR = LEADING_LEDGER
        I_BUKRS = OS_BKPF-BUKRS
        I_BELNR = OS_BKPF-BELNR
        I_GJAHR = OS_BKPF-GJAHR
        I_BSTAT = OS_BKPF-BSTAT
      IMPORTING
        ET_BSEG = IT_BSEG[]
      EXCEPTIONS
        NOT_FOUND = 1
        OTHERS    = 2.

    READ TABLE IT_BSEG
      INTO OS_BSEG
      WITH KEY BUKRS = P_SALID
               BELNR = IV_BELNR
               GJAHR = IV_GJAHR
               BUZEI = IV_BUZEI.
  ENDIF.
ENDIF.
```

> Nếu điều kiện lấy `BSEG` đã có sẵn thông tin `BKPF`, không cần `SELECT BKPF` lại; dùng structure `BKPF` đã取得済み.

### Table retrieval

#### Cú pháp cũ — table (旧)

```abap
SELECT BUKRS
       GJAHR
       BELNR
       BUZEI
       ...
  INTO TABLE OT_BSEG
  FROM BSEG
  FOR ALL ENTRIES IN IT_BSIS
  WHERE BUKRS = IT_BSIS-BUKRS
    AND GJAHR = IT_BSIS-GJAHR
    AND BELNR = IT_BSIS-BELNR
    AND BUZEI = IT_BSIS-BUZEI.
```

#### Cú pháp mới — table (新)

```abap
SELECT BUKRS
       GJAHR
       BELNR
       BUZEI
       ...
  INTO TABLE OT_BSEG
  FROM BSEG
  FOR ALL ENTRIES IN IT_BSIS
  WHERE BUKRS = IT_BSIS-BUKRS
    AND GJAHR = IT_BSIS-GJAHR
    AND BELNR = IT_BSIS-BELNR
    AND BUZEI = IT_BSIS-BUZEI.

SELECT BUKRS
       BELNR
       GJAHR
       BSTAT
  FROM BKPF
  INTO CORRESPONDING FIELDS OF TABLE LT_BKPF
  FOR ALL ENTRIES IN IT_BSIS
  WHERE BUKRS = IT_BSIS-BUKRS
    AND GJAHR = IT_BSIS-GJAHR
    AND BELNR = IT_BSIS-BELNR
    AND BSTAT = 'U'.

LOOP AT LT_BKPF INTO DATA(LS_BKPF).

  LEADING_LEDGER = CL_FINS_ACDOC_UTIL=>GET_LEADING_LEDGER( ).

  CALL FUNCTION 'FAGL_GET_GL_DOCUMENT'
    EXPORTING
      I_RLDNR = LEADING_LEDGER
      I_BUKRS = LS_BKPF-BUKRS
      I_BELNR = LS_BKPF-BELNR
      I_GJAHR = LS_BKPF-GJAHR
      I_BSTAT = LS_BKPF-BSTAT
    IMPORTING
      ET_BSEG = LT_BSEG_ACDOC
    EXCEPTIONS
      NOT_FOUND = 1
      OTHERS    = 2.

  IF SY-SUBRC = 0.
    LOOP AT LT_BSEG_ACDOC INTO DATA(LS_BSEG_ACDOC)
      WHERE BUKRS = LS_BKPF-BUKRS
        AND BELNR = LS_BKPF-BELNR
        AND GJAHR = LS_BKPF-GJAHR
        AND BUZEI = IT_BSIS-BUZEI.

      APPEND LS_BSEG_ACDOC TO OT_BSEG.
    ENDLOOP.
  ENDIF.

ENDLOOP.
```
