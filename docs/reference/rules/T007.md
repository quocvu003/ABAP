## T007 - Table `MAEX` không còn lưu dữ liệu

* **ID:** `T007`
* **Nội dung lỗi (エラー内容):** Table `MAEX` không còn lưu dữ liệu trên S/4HANA.
* **Cách khắc phục (対応方法):** Lấy dữ liệu thay thế từ `/SAPSLL/MARLRG`. Nếu item/field cần dùng không tồn tại trong `/SAPSLL/MARLRG`, cần lấy từ nguồn khác theo thiết kế hoặc confirm Teamlead.

### Lưu ý quan trọng

* `MAEX` là table quản lý cũ theo MM cho Export Control.
* `/SAPSLL/MARLRG` là nguồn thay thế trên S/4HANA, nhưng **cấu trúc field không hoàn toàn giống `MAEX`**.
* Không thay trực tiếp `SELECT * FROM MAEX ... INTO TABLE LT_MAEX` thành `SELECT * FROM /SAPSLL/MARLRG ... INTO TABLE LT_MAEX`.
* Phải chọn các field thật sự cần dùng và map lại sang structure/table output của chương trình.
* Mapping field đã xác nhận:
  * `MAEX-ALNUM` -> `/SAPSLL/MARLRG-CONCL`
  * `MAEX-EMBGR` -> `/SAPSLL/MARLRG-CONGR`
* Luôn ưu tiên cú pháp ABAP mới, dùng host variable `@` trong Open SQL.

### Cú pháp cũ (旧)

```abap
SELECT *
  FROM MAEX
  WHERE MATNR = LV_MATNR
  INTO TABLE LT_MAEX.
```

### Cú pháp mới (新)

```abap
* S/4HANA Migration UPD START
*  SELECT *
*    FROM MAEX
*    WHERE MATNR = LV_MATNR
*    INTO TABLE LT_MAEX.
  SELECT MATNR,
         CONCL,
         CONGR
    FROM /SAPSLL/MARLRG
    WHERE MATNR = @LV_MATNR
    INTO TABLE @DATA(LT_MARLRG).
* S/4HANA Migration UPD END
```

### Khi chương trình cũ cần giữ output/table kiểu cũ

Nếu logic phía sau vẫn dùng internal table kiểu `MAEX` hoặc custom structure tương đương, cần map lại field tương ứng. Ví dụ:

```abap
* S/4HANA Migration ADD START
DATA(LT_MAEX_NEW) = VALUE TT_MAEX(
  FOR LS_MARLRG IN LT_MARLRG
  ( MATNR = LS_MARLRG-MATNR
    ALNUM = LS_MARLRG-CONCL
    EMBGR = LS_MARLRG-CONGR ) ).
* S/4HANA Migration ADD END
```

> `TT_MAEX` chỉ là ví dụ tên table type. Khi sửa thực tế, dùng đúng table type/internal table hiện có của chương trình.
