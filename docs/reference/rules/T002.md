## T002 - Table `KONV` được thay bằng CDS view `V_KONV_CDS`

* **ID:** `T002`
* **Nội dung lỗi (エラー内容):** Trên S/4HANA, condition table `KONV` không còn được truy vấn trực tiếp; phải đọc qua CDS view `V_KONV_CDS`.
* **Cách khắc phục (対応方法):** Thay `FROM KONV` bằng `FROM V_KONV_CDS`. Nếu câu lệnh có JOIN hoặc các đoạn code sau vẫn reference `KONV-*` (alias hoặc field qualifier), dùng alias `V_KONV_CDS AS KONV` để giữ nguyên các reference còn lại.

### Nguyên tắc xử lý

* `V_KONV_CDS` có field list tương ứng `KONV`; field name không đổi (`KNUMH`, `KBETR`, `KWERT`, ...). Không cần map lại từng field.
* `SELECT *` được phép giữ nguyên, chỉ thay `FROM KONV` → `FROM V_KONV_CDS`. Không cần liệt kê lại field list khi chuyển sang CDS view.
* Khi câu lệnh **có JOIN** hoặc trong cùng câu SELECT có nhiều chỗ qualify field bằng `KONV-...` / `KONV~...`, **phải dùng alias** `V_KONV_CDS AS KONV` để các reference khác không phải sửa theo. Khi câu lệnh **không có JOIN** và không dùng qualifier `KONV-...`, không cần alias.
* Ưu tiên cú pháp ABAP mới, dùng host variable `@` trong Open SQL khi có thể (xem [[prefer-explicit-code]]). Tuy nhiên, nếu task chỉ là thay table thì giữ nguyên cú pháp gốc trừ phần `FROM`, không refactor thêm.
* Không tự thêm pseudo-comment cho `FROM KONV` — fix bằng cách thay nguồn dữ liệu, không phải bằng `"#EC`.

### Cú pháp cũ (旧) — không JOIN


### Cú pháp mới (新) — khi có JOIN hoặc qualifier `KONV-...`

Khi câu lệnh có JOIN với table khác, hoặc các đoạn code phía sau vẫn dùng `KONV-FIELD` để qualify, dùng alias `V_KONV_CDS AS KONV`:

```abap

*  FROM KONV
  FROM V_KONV_CDS AS KONV       "結合時は V_KONV_CDS AS KONV
* S/4HANA Migration UPD END
    INNER JOIN VBRK ON KONV~KNUMV = VBRK~KNUMV
  WHERE KONV~KSCHL = 'PR00'.
```

> Alias chỉ thêm khi thực sự cần (có JOIN / có qualifier). Câu SELECT đơn giản không qualifier thì không thêm alias để tránh đổi không cần thiết.
